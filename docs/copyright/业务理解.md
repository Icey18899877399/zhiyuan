# 业务理解：智源—高校智能信息服务平台

> 本文件用于在撰写申请表和操作手册之前固定项目的业务口径。所有后续材料中
> 的行业、用户、功能描述都以此处为准。

## 1. 产品定位

智源是一个面向高校的智能信息服务平台。它把分散在学校教务系统、学院网站、
就业网和学院公众号上的校园通知，**通过定时调度的爬虫自动聚合到统一数据库**，
并通过大语言模型按学生关心的话题做问答。

平台两大技术特点：

1. **动态 RAG**：通知入库即同步生成向量（pgvector），检索时按"语义相似度
   × 时间衰减"加权排序——既匹配语义又偏好新内容，避免传统关键词检索把过期
   通知排在最前。
2. **按话题分流**：5 个 agent 标签（学业 / 活动 / 党团 / 就业 / 行政）对应
   5 类校园场景；前端选定标签即在该话题分类内做检索，避免跨话题串话。

学生不再需要分别打开教务网、学院网站、就业网和公众号手动翻找通知。打开"智源"
对话页，选择对应的话题分类，用一句口语化的问话就能拿到回答以及参考来源链接。

## 2. 面向领域 / 行业

高等教育信息化 / 校园智能服务 / 智能问答应用。

子领域：高校教务与学生事务、校园活动与文化、党团工作、就业指导、行政事务。

## 3. 目标用户

- **本科生 / 研究生**：日常获取学业、活动、就业、党团信息的主要用户
- **辅导员 / 班主任**：批量了解面向学生发布的通知，便于答疑和工作部署
- **学生组织成员**：跟进党团活动、社团赛事、招新等信息
- **学校信息化部门 / 学院管理员**：把已有的多源通知统一展示，减少重复发布

## 4. 核心价值

- **聚合**：APScheduler 每 30 分钟自动巡查三个公开数据源（教务处、计算机
  与网络空间安全学院、就业网）；URL 与 SHA-256 内容哈希双层去重，
  保证不重复入库
- **分类**：抓取时按关键词自动归入"学业 / 活动 / 党团 / 就业 / 其他"
  五类，方便按场景使用
- **向量化**：每条新通知入库时同步生成 1024 维 embedding，存入
  pgvector 列；问答检索改用余弦相似度而非关键词，对同义不同字的提问
  能命中相关通知
- **时间衰减**：检索排序公式 `score = cos_sim × exp(-λ × age_days)`，
  λ 默认 0.05（半衰期约 14 天）；同等相关性下越新的通知排名越靠前
- **可追溯**：每条问答的回答都标注参考来源，用户可一键打开原文核对；
  每次爬取/问答都落 crawl_runs / chat_logs 流水
- **可降级**：embedding 不可用时（无 API Key / 接口异常）retrieval 静默
  回退到关键词 ILIKE + 中文 bi-gram，保证服务连续

## 5. 主要业务功能

按学生使用顺序排列：

1. **对话中心**：在"学业 / 活动 / 党团 / 就业 / 行政"五个话题标签里选择，
   输入问题获得回答，并查看本次回答引用的通知列表
2. **RAG 知识库展示**：以分类网格的形式浏览当前知识库内的所有通知，
   点击分类卡片查看该分类下的文章列表
3. **身份设置**：保存学院、年级、身份角色和补充标签，便于后续个性化推荐
4. **每日数据**：实时展示当日各 spider 巡查次数、新增通知数、问答数；
   底部"知识库更新日志"按时间倒序展示最近 20 次跑批流水
5. **后台定时爬取**：APScheduler 每 30 分钟自动巡查三个数据源，
   每次跑批写一条 CrawlRun 流水；支持手动命令行触发
6. **后台分类**：抓取入库时按关键词规则自动判定话题分类
7. **后台 embedding 同步**：每条新通知入库时通过 embedding provider
   生成 1024 维向量；旧数据通过 backfill 脚本幂等回填
8. **后台问答日志**：每次问答自动记录问题、回答和命中文章列表，
   便于后续优化

## 6. 典型操作流程

以最常见的"查询学业通知"为例：

1. 用户打开浏览器访问平台首页
2. 默认进入对话中心，停留在"学业 agent"标签
3. 用户在输入框输入一句问题，例如"2026 年辅修招生录取名单出来了吗"
4. 系统对问题先调用 embedding 服务生成向量；
   在数据库中以 pgvector 余弦距离与"学业"分类做向量检索，
   并叠加时间衰减加权排序，召回最相关的若干条通知
5. 系统把召回的通知摘要作为上下文，调用 DeepSeek 模型生成回答
6. 页面上显示模型的回答，并在下方折叠显示参考来源列表
7. 用户点击参考来源链接，跳转到学校原文页面核对完整通知

如果用户更关心活动信息，可以切换到"活动 agent"标签后再提问，系统会改用
"活动"分类做检索。RAG 知识库展示页提供了不依赖问答的浏览路径，
适合用户想要按分类批量浏览所有通知。

## 8. 证据来源

- `app/scheduler.py`：APScheduler 定时调度，三个 spider 每 30 分钟轮一次
- `app/models/crawl_run.py`、`app/api/stats.py`：跑批流水模型 + 统计接口
- `app/crawler/base.py`：去重、入库、CrawlRun 流水写入、embedding 同步钩子
- `app/crawler/spiders/cuc_jwc_notice.py`、`cuc_cs_notice.py`、`cuc_career.py`：
  教务、计算机学院、就业网三个数据源的实际爬取实现
- `app/crawler/classifier.py`：五类话题的关键词规则
- `app/services/embedding.py`：智谱 / OpenAI 多 provider embedding 客户端
- `app/services/retrieval.py`：向量检索主路径 + 关键词 fallback；时间衰减
  打分函数 `time_decay_score`
- `app/api/chat.py`：检索 + DeepSeek 调用 + 写问答日志的完整调用链
- `app/api/articles.py`：分类聚合、文章列表、详情接口
- `frontend/chat.js`、`frontend/rag.js`、`frontend/daily.js`：前端三类核心交互
- `frontend/index.html`、`rag.html`、`identity.html`、`daily.html`：四个页面的真实结构
- `alembic/versions/b2c3d4e5f6a7_pgvector_embedding.py`：启用 pgvector
  扩展、加 embedding 列、建 IVFFLAT 索引
