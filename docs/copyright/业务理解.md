
# 业务理解：智源—高校智能信息服务平台

> 本文件用于在撰写申请表和操作手册之前固定项目的业务口径。所有后续材料中的行业、用户、功能描述都以此处为准。

## 1. 产品定位

智源是一个面向高校的智能信息服务平台。它把分散在学校教务系统、学院通知、就业网站和学院公众号上的校园通知，统一聚合到一个数据库中，并通过大语言模型按学生关心的话题做问答。

学生不再需要分别打开教务网、学院网站、就业网和公众号手动翻找通知。打开"智源"对话页，选择对应的话题分类，用一句口语化的问话就能拿到回答以及参考来源链接。

## 2. 面向领域 / 行业

高等教育信息化 / 校园智能服务 / 智能问答应用。

子领域：高校教务与学生事务、校园活动与文化、党团工作、就业指导。

## 3. 目标用户

- **本科生 / 研究生**：日常获取学业、活动、就业、党团信息的主要用户
- **辅导员 / 班主任**：批量了解面向学生发布的通知，便于答疑和工作部署
- **学生组织成员**：跟进党团活动、社团赛事、招新等信息
- **学校信息化部门 / 学院管理员**：把已有的多源通知统一展示，减少重复发布

## 4. 核心价值

- **聚合**：自动从教务处、计算机与网络空间安全学院、就业网三个网站抓取通知，未来支持学院公众号；统一入库去重
- **分类**：抓取时按关键词自动归入"学业 / 活动 / 党团 / 就业 / 其他"五类，方便按场景使用
- **问答**：基于真实通知做检索增强问答，回答附带原文链接，避免大模型凭空生成
- **可追溯**：每条问答的回答都标注参考来源，用户可一键打开原文核对

## 5. 主要业务功能

按学生使用顺序排列：

1. **对话中心**：在"学业 / 活动 / 党团 / 就业 / 行政"五个话题标签里选择，输入问题获得回答，并查看本次回答引用的通知列表
2. **RAG 知识库展示**：以分类网格的形式浏览当前知识库内的所有通知，点击分类卡片查看该分类下的文章列表
3. **身份设置**：保存学院、年级、身份角色和补充标签，便于后续个性化推荐
4. **每日数据**：按日期查看各话题的请求次数统计与知识库更新日志
5. **后台爬取**：定时从教务处、计算机学院、就业网抓取通知，支持手动触发
6. **后台分类**：抓取入库时自动判定话题分类
7. **后台问答日志**：每次问答自动记录问题、回答和命中文章列表，便于后续优化

## 6. 典型操作流程

以最常见的"查询学业通知"为例：

1. 用户打开浏览器访问平台首页
2. 默认进入对话中心，停留在"学业 agent"标签
3. 用户在输入框输入一句问题，例如"2026 年辅修招生录取名单出来了吗"
4. 系统将问题与"学业"分类的通知库做关键词检索，召回最相关的若干条
5. 系统把召回的通知摘要作为上下文，调用 DeepSeek 模型生成回答
6. 页面上显示模型的回答，并在下方折叠显示参考来源列表
7. 用户点击参考来源链接，跳转到学校原文页面核对完整通知

如果用户更关心活动信息，可以切换到"活动 agent"标签后再提问，系统会改用"活动"分类做检索。RAG 知识库展示页提供了不依赖问答的浏览路径，适合用户想要按分类批量浏览所有通知。

## 8. 证据来源

- `app/crawler/spiders/cuc_jwc_notice.py`、`cuc_cs_notice.py`、`cuc_career.py`：教务、计算机学院、就业网三个数据源的实际爬取实现
- `app/crawler/classifier.py`：五类话题的关键词规则
- `app/api/chat.py`：检索 + DeepSeek 调用 + 写问答日志的完整调用链
- `app/api/articles.py`：分类聚合、文章列表、详情接口
- `frontend/chat.js`、`frontend/rag.js`、`frontend/daily.js`：前端三类核心交互
- `frontend/index.html`、`rag.html`、`identity.html`、`daily.html`：四个页面的真实结构
