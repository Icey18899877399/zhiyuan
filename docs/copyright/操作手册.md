## 第一章 软件概述

### 1.1 软件简介

智源是一款面向高校师生的校园信息汇集与智能问答平台。学校的通知信息分布在
教务处网站、各学院网站、就业网和学院公众号，学生在了解某项政策或查找具体
通知时，往往需要在几个网站之间反复切换并手动检索，体验欠佳。智源通过定时
调度的爬虫每 30 分钟自动巡查三个数据源，把通知统一抓取入库；新通知入库
时同步生成 1024 维语义向量（pgvector），问答时按"余弦相似度 × 时间衰减
加权"做检索，最后以五个话题标签（学业、活动、党团、就业、行政）的方式
重新组织，使学生可以在一个对话界面中按场景直接提问，并能在回答下方看到
本次回答引用了哪些原始通知。

平台同时提供一个分类浏览界面。学生不必一定提问，也可以打开"RAG 知识库
展示"页，按话题查看当前知识库中已经收录的通知，并直接跳转到学校原文核对。

### 1.2 适用对象

智源主要面向以下几类用户：

- 在校本科生与研究生：用于日常获取学业、活动、就业、党团信息
- 辅导员与班主任：用于了解面向学生发布的通知，便于答疑和工作部署
- 学生组织成员：用于跟进党团活动、社团赛事和招新等信息
- 学校信息化部门与学院管理员：把多个网站发布的通知统一展示，减少重复发布

### 1.3 运行环境

软件运行依赖以下环境：

- 操作系统：Windows 10/11、macOS 13 及以上版本，或主流 Linux 发行版
- Python 3.11 及以上版本
- PostgreSQL 16 + pgvector 扩展（推荐用 `pgvector/pgvector:pg16` 镜像）
- Docker 与 Docker Compose（用于一键启动数据库）
- 现代浏览器（Chrome、Edge、Safari、Firefox 任一最新版本）
- DeepSeek 大语言模型 API（在 https://platform.deepseek.com 申请）
- 智谱 BigModel embedding API（可选，在 https://open.bigmodel.cn 申请；
  不配置则检索自动降级到关键词模式）

### 1.4 名词约定

为方便后续章节理解，先对几个词语做约定：

- **话题**：指学业、活动、党团、就业、行政五个分类标签，对应数据库中
  的 category 字段
- **知识库**：指系统通过爬虫从各个来源抓取并入库的全部校园通知
- **参考来源**：每次问答时，被检索到并参与回答生成的若干条通知
- **后台爬取**：指由后端定时任务（30 分钟一次）或手动命令执行的通知抓取
- **跑批流水**：每次爬虫执行写入 crawl_runs 表的一条记录，包含开始/结束
  时间、状态、抓取数、新增数、失败数
- **embedding**：把一段文本编码为 1024 维浮点向量，用于在数据库中做语义检索
- **时间衰减**：按通知发布时间到当前的天数做指数衰减，使较新的通知排名靠前

## 第二章 运行环境与启动

### 2.1 数据库启动

平台使用 PostgreSQL 16 + pgvector 扩展存储所有通知、用户、问答日志、爬虫
流水以及向量数据。项目根目录的 docker-compose.yml 已绑定 pgvector 镜像，
一行命令即可启动。

打开命令行，进入项目根目录，执行：

```
docker compose up -d
```

执行后会拉取 pgvector/pgvector:pg16 镜像并启动容器。容器进入健康状态后，
数据库会监听本机的 5432 端口，账号 zhiyuan、密码 zhiyuan_dev_pwd、库名
zhiyuan。

【截图预留：请在此处插入"docker compose up -d 启动后 docker compose ps
显示 healthy"截图。】

### 2.2 安装依赖

在项目根目录执行：

```
pip install -e .
```

会安装 FastAPI、SQLAlchemy、asyncpg、psycopg2-binary、pgvector、httpx、
APScheduler、OpenAI SDK 等依赖。

### 2.3 数据表创建

数据库容器启动后，需要执行数据库迁移以创建所有表与启用 pgvector：

```
alembic upgrade head
```

执行后会依次跑三次迁移：

1. `bd3a595058f1` 初始化 articles / users / chat_logs / user_unread 四张表
2. `a1b2c3d4e5f6` 新增 crawl_runs 跑批流水表（M1）
3. `b2c3d4e5f6a7` 启用 pgvector 扩展、articles 加 embedding vector(1024) 列、
   建立 IVFFLAT 余弦索引（M2）

【截图预留：请在此处插入"alembic upgrade head 输出三条 INFO"截图。】

### 2.4 后端启动

数据库就绪后，启动 Web 服务。在项目根目录执行：

```
uvicorn app.main:app --reload
```

启动日志中会先看到"挂载前端静态目录"的提示，说明前端页面被挂到了同一个
端口上。随后看到"zhiyuan-backend 启动"、"已注册定时任务"、"调度器已启动"
和"Uvicorn running on http://127.0.0.1:8000"几条字样，表示服务可以访问且
定时爬虫已就绪。

【截图预留：请在此处插入"uvicorn 启动日志"截图。】

### 2.5 浏览器访问

在浏览器地址栏输入：

```
http://localhost:8000
```

回车后即可进入对话中心首页。

### 2.6 必填配置

在第一次启动之前，需要在项目根目录的 .env 文件中填写：

1. **DEEPSEEK_API_KEY**（必填）：问答功能调用大语言模型所必需的密钥，
   缺失时问答接口返回错误提示"未配置 DEEPSEEK_API_KEY"
2. **EMBEDDING_PROVIDER + ZHIPU_API_KEY**（可选）：选择向量化服务；
   设为 `disabled` 时 retrieval 自动降级到关键词检索，无需任何 embedding key
3. **SCHEDULER_ENABLED**（可选）：默认 true；本地纯前端开发时可设为 false
   关闭自动爬取

页面底部是输入框与发送按钮。用户可以在输入框中键入问题，回车或点击"发送"

系统消息以"正在思考中..."占位，等待后端返回回答后再替换为真实内容。

## 第四章 对话问答

### 4.1 提一个学业问题

以最常见的"查询学业通知"为例。用户保持在"学业 agent"标签，输入框中输入
一句问题，例如"2026 年辅修招生录取名单出来了吗"，回车或点击发送。

系统首先把问题文本送给 embedding 服务编码成 1024 维向量，
然后在数据库中按"学业"分类做向量检索，得到余弦距离最近的若干条通知，
再按时间衰减重新加权排序，最终把这些通知的标题与摘要作为上下文，
调用 DeepSeek 模型生成一段自然语言回答。回答正文显示在系统消息气泡中，
前面有一个方括号包裹的话题标签例如"[学业]"，便于用户在多轮对话时快速
识别。

【截图预留：请在此处插入"学业 agent 下提问并获得回答"截图。】

### 4.2 查看参考来源

每条系统回答的下方有一个"参考来源 (n)"折叠区，n 表示本次回答用到了多少条
通知。点击折叠区可以展开，列出每条参考通知的标题、来源标签和原文链接。
点击链接会在新标签页打开学校原文页面，便于用户核对完整通知。

参考来源列表是问答的可追溯性保障。如果系统的回答与用户的预期不符，用户
可以通过参考来源直接对照学校原文，判断是检索结果偏题还是模型理解偏差。

【截图预留：请在此处插入"展开参考来源折叠区"截图。】

### 4.3 切换话题继续提问

用户可以随时点击其他话题标签切换。切换后对话窗口不会清空，新提问的回答
会写入同一个对话流。例如先在学业 agent 下问辅修通知，再切到就业 agent
问"计算机学院最近有招聘吗"，系统会自动改用"就业"分类的通知做检索。

### 4.4 没有相关通知时的反馈

如果用户提问的内容在当前话题分类下没有任何相关通知，例如在党团 agent
下问"如何申请助学金"，系统会明确告知"目前知识库中没有相关内容"，并建议
用户切换话题或直接查阅学校官网，不会编造不存在的政策。这是平台的设计
原则之一：宁可如实告诉用户没有信息，也不允许模型臆造校园政策。

### 4.5 异常情况

如果发送提问后长时间没有收到回答，常见原因有：

- DeepSeek API Key 未配置或已欠费：系统会返回"未配置 DEEPSEEK_API_KEY"
- 网络异常导致大模型接口超时：系统会返回"大模型调用失败"并附带具体错误
- embedding provider 异常：retrieval 会**自动降级**到关键词检索，
  对用户透明不会报错
- 后端服务未启动：浏览器会显示"服务暂时不可用"

文章"的提示，这种情况通常意味着对应的爬虫还未执行或该来源最近没有发布
新通知。

## 第六章 身份设置

### 6.1 设置目的

平台支持用户保存一组基本身份信息，包括所在学院、所在年级、身份角色和补充
标签。这些信息保存在浏览器本地存储中，便于后续平台根据身份做个性化推荐
与展示。

### 6.2 填写步骤

用户点击左侧导航栏的"身份设置"，进入身份设置页。页面提供一个简单的表单：

1. 学院：从下拉框选择，例如"计算机学院"、"人工智能学院"、"数据科学学院"
2. 年级：从下拉框选择，例如"2024 级"
3. 身份角色：从下拉框选择，例如"本科生"、"研究生"、"辅导员"、"教师"
4. 标签补充：在文本框中填写自定义标签，例如"转专业、考研、学生干部"

填写完成后点击"保存身份标签"按钮，页面下方会显示"身份标签已保存，可用于
后续对话 RAG 检索"。

【截图预留：请在此处插入"身份设置页填写后并保存"截图。】

### 6.3 数据存储说明

身份信息仅保存在用户当前浏览器的本地存储中，不会上传到服务器，也不会跨
设备同步。用户清理浏览器数据或更换浏览器后需要重新填写。

## 第七章 每日数据查看

### 7.1 用途

平台维护两份按日聚合的统计数据，用于让管理员或运营人员评估"动态爬取"的
实际运行节奏：

1. **当日概览**：今日各 spider 巡查次数、成功/失败比、新增通知总数、
   分类拆分（如学业 X / 活动 Y）、问答数
2. **分爬虫详情**：每个数据源的巡查次数、成功失败次数、抓取条目、新增入库
3. **知识库更新日志**：按时间倒序列出最近 20 次跑批流水，每行带状态点
   🟢 / 🔴 / 🟡

### 7.2 操作步骤

用户点击左侧导航栏的"每日数据"，进入每日数据页。页面顶部是一个日期选择器
和"查看"按钮，下方是"当日概览"卡片网格、"分爬虫巡查详情"表格、"知识库更新
日志"列表。

操作流程：

1. 在日期选择器中选定一个日期
2. 点击"查看"按钮
3. 页面下方的三块内容同时刷新为该日期下的真实数据

【截图预留：请在此处插入"每日数据页选择日期并展示当日概览与跑批流水"截图。】

### 7.3 与底层接口的对应

页面底层调用两个后端接口：

- `GET /api/stats/daily?date=YYYY-MM-DD`：返回当日聚合统计
- `GET /api/stats/crawl-runs?limit=20`：返回最近 N 次跑批流水

这两个接口聚合的是真实写入 crawl_runs 表的数据，没有任何 mock。

## 第八章 后台运营

本章面向系统管理员，介绍如何运维平台的后台数据。

### 8.1 自动定时巡查

平台后端启动时，APScheduler 调度器会自动注册三个定时任务，每 30 分钟一次
分别触发 cuc_jwc_notice、cuc_cs_notice、cuc_career 三个 spider。每次跑批
写入一条 CrawlRun 流水，包含开始/结束时间、status、抓取数、新增数、跳过数、
失败数。

如果只想关掉自动巡查（例如本地纯前端开发），可以在 .env 设置：

```
SCHEDULER_ENABLED=false
```

然后重启 uvicorn。

### 8.2 手动触发爬虫

管理员在项目根目录命令行执行：

```
python -m scripts.run_crawler cuc_jwc_notice --max 20
python -m scripts.run_crawler cuc_cs_notice --max 20
python -m scripts.run_crawler cuc_career --max 20
```

每个命令会从对应来源抓取若干页通知。命令结束时会输出统计结果，包括
total（抓取总数）、inserted（新增数）、skipped（已存在跳过数）、
errors（错误数）。

【截图预留：请在此处插入"执行爬虫命令并看到完成统计"截图。】

### 8.3 回填历史数据的 embedding

对于在配置 embedding provider 之前已经入库的旧数据，运行以下命令将其逐条
补齐 embedding：

```
python -m scripts.backfill_embeddings --batch 32
```

脚本是幂等的：只处理 embedding IS NULL 的行，中途崩溃再跑也不会重复消耗
API 配额。

### 8.4 查看数据库内容

管理员可以通过 Docker 进入 PostgreSQL 容器，使用 psql 客户端查看数据库
内容：

```
docker exec -it zhiyuan-postgres psql -U zhiyuan -d zhiyuan
```

进入交互式终端后，可以执行 SQL 查询，例如：

```
SELECT category, COUNT(*) FROM articles GROUP BY category;
SELECT source, COUNT(*) FROM crawl_runs WHERE started_at >= NOW() - INTERVAL '1 day'
  GROUP BY source;
SELECT COUNT(*) AS total, COUNT(embedding) AS with_embedding FROM articles;
```

第一条查询统计各话题下通知数量；第二条统计过去 24 小时各 spider 的跑批
次数；第三条统计 embedding 列的覆盖率。

## 第九章 注意事项与常见问题

### 9.1 API Key 配置

DeepSeek API Key 是问答功能的必备条件。如果浏览器对话页提示"未配置
DEEPSEEK_API_KEY"，管理员需要：

1. 打开项目根目录的 .env 文件
2. 在 DEEPSEEK_API_KEY= 后面填入完整的密钥
3. 保存文件
4. 重启 uvicorn 后端服务

### 9.2 embedding 切换

`.env` 的 `EMBEDDING_PROVIDER` 字段控制向量化服务：

- `disabled`：完全关闭，retrieval 只用关键词检索
- `zhipu`：使用智谱 BigModel embedding-3，需要 `ZHIPU_API_KEY`
- `openai`：使用 OpenAI text-embedding-3-small（兼容口径）

切换 provider 后建议重启 uvicorn，并跑一次 `backfill_embeddings.py` 让
新 provider 的向量风格统一。

### 9.3 数据库连接

如果命令行执行 alembic upgrade head 时报错"Connection refused"，通常是
PostgreSQL 容器没有启动或还没进入健康状态。管理员可以执行 docker compose
ps 查看容器状态，确认 zhiyuan-postgres 显示 healthy 后再重试。

如果报错"extension vector does not exist"，说明镜像还是老版 postgres，
不是 pgvector 版本。需要 `docker compose down && docker compose up -d`
重新拉 pgvector/pgvector:pg16 镜像。

### 9.4 跨域访问

平台默认放开了所有来源的跨域访问，方便开发调试。生产部署时需要把 .env
中的 CORS_ORIGINS 改为具体的域名列表，避免不必要的安全风险。

### 9.5 浏览器缓存

修改前端文件后如果浏览器看到的还是旧版本，可以使用 Ctrl+F5 进行强制刷新，
绕过浏览器缓存。

### 9.6 爬虫被反爬

如果某个爬虫一直返回 0 条数据，通常是目标网站做了反爬限制或站点改版。
管理员可以等待一段时间后重试，或者更新 spider 中的 User-Agent / URL 模板。
