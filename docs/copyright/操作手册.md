## 第一章 软件概述

### 1.1 软件简介

智源是一款面向高校师生的校园信息汇集与智能问答平台。学校的通知信息分布在教务处网站、各学院网站、就业网和学院公众号，学生在了解某项政策或查找具体通知时，往往需要在几个网站之间反复切换并手动检索，体验欠佳。智源把这些来源的通知统一抓取到一个数据库中，并以五个话题标签的方式重新组织，使学生可以在一个对话界面中按照"学业、活动、党团、就业、行政"五个角度直接提问，并能在回答下方看到本次回答引用了哪些原始通知。

平台同时提供一个分类浏览界面。学生不必一定提问，也可以打开"RAG 知识库展示"页，按话题查看当前知识库中已经收录的通知，并直接跳转到学校原文核对。

### 1.2 适用对象

智源主要面向以下几类用户：

- 在校本科生与研究生：用于日常获取学业、活动、就业、党团信息
- 辅导员与班主任：用于了解面向学生发布的通知，便于答疑和工作部署
- 学生组织成员：用于跟进党团活动、社团赛事和招新等信息
- 学校信息化部门与学院管理员：把多个网站发布的通知统一展示，减少重复发布

### 1.3 运行环境

软件运行依赖以下环境：

- 操作系统：Windows 10/11、macOS 13 及以上版本，或主流 Linux 发行版
- Python 3.11 及以上版本
- PostgreSQL 16 数据库
- Docker 与 Docker Compose（用于一键启动数据库）
- 现代浏览器（Chrome、Edge、Safari、Firefox 任一最新版本）
- DeepSeek 大语言模型 API（在 https://platform.deepseek.com 申请）

### 1.4 名词约定

为方便后续章节理解，先对几个词语做约定：

- **话题**：指学业、活动、党团、就业、行政五个分类标签，对应数据库中的 category 字段
- **知识库**：指系统通过爬虫从各个来源抓取并入库的全部校园通知
- **参考来源**：每次问答时，被检索到并参与回答生成的若干条通知
- **后台爬取**：指由后端定时任务或手动命令执行的通知抓取动作

## 第二章 运行环境与启动

### 2.1 数据库启动

平台使用 PostgreSQL 16 存储所有通知、用户、问答日志数据。项目根目录提供了 docker-compose.yml，可通过一行命令启动数据库。

打开命令行，进入项目根目录，执行：

```
docker compose up -d
```

执行后会看到 Docker 拉取 postgres:16-alpine 镜像并启动容器。容器进入健康状态后，数据库会监听本机的 5432 端口，账号 zhiyuan、密码 zhiyuan_dev_pwd、库名 zhiyuan。

【截图预留：请在此处插入"docker compose up -d 启动后 docker compose ps 显示 healthy"截图。】

### 2.2 数据表创建

数据库容器启动后，需要执行一次数据库迁移以创建所有表。在项目根目录执行：

```
alembic upgrade head
```

执行后会看到 alembic 输出迁移版本号。命令完成后，articles、users、user_unread、chat_logs 四张表创建完毕，分别用于存放通知、用户、用户未读关系、问答日志。

【截图预留：请在此处插入"alembic upgrade head 输出"截图。】

### 2.3 后端启动

数据库就绪后，启动 Web 服务。在项目根目录执行：

```
uvicorn app.main:app --reload
```

启动日志中会先看到"挂载前端静态目录"的提示，说明前端页面被挂到了同一个端口上。随后看到"zhiyuan-backend 启动"和"Uvicorn running on http://127.0.0.1:8000"的字样，表示服务可以访问。

【截图预留：请在此处插入"uvicorn 启动日志"截图。】

### 2.4 浏览器访问

在浏览器地址栏输入：

```
http://localhost:8000
```

回车后即可进入对话中心首页。

### 2.5 必填配置

在第一次启动之前，需要在项目根目录的 .env 文件中至少填写一项：DeepSeek API Key。这是问答功能调用大语言模型所必需的密钥，缺失时问答接口会返回错误提示"未配置 DEEPSEEK_API_KEY，无法调用大模型"。

## 第三章 进入软件与界面布局

### 3.1 首次访问

浏览器打开平台地址后，默认进入对话中心。页面左侧是固定的导航栏，列出了四个功能入口：RAG 知识库展示、身份设置、每日数据、对话。其中"对话"在初次进入时是选中状态。

【截图预留：请在此处插入"首次访问对话中心的整体布局"截图。】

### 3.2 顶部信息区

页面顶部左侧是平台名称"对话中心"和一个圆点状态指示；右侧是"切换深色模式"按钮。点击该按钮，整个页面会切换到深色配色，并且这个选择会保存在浏览器本地，下一次访问会自动恢复用户上次的偏好。

【截图预留：请在此处插入"深色模式效果"截图。】

### 3.3 话题标签

页面中段是一排话题标签按钮：学业 agent、活动 agent、党团 agent、就业 agent、行政 agent。每个标签都对应一类校园通知。点击任一标签后，下方输入框的占位文字会切换为对应话题的引导语，例如学业 agent 显示"你想了解哪些课程安排或学业政策"，活动 agent 显示"你想了解什么方面的校园活动"。

切换标签会决定接下来的提问把哪一类通知作为检索范围。在学业 agent 状态下提问，系统只会从被归类为"学业"的通知里挑选参考资料；切换到活动 agent 后再提问，检索范围切换为"活动"分类。

【截图预留：请在此处插入"切换不同 agent 标签后输入框引导语变化"截图。】

### 3.4 对话窗口

标签下方是对话窗口。窗口默认显示一条欢迎语，例如"你好，我是学业 Agent，你可以问我课程安排、学分、培养方案等问题"。窗口支持垂直滚动，历史问答会按时间顺序由上至下排列。用户的提问以蓝色背景显示在右侧，系统的回答以绿色背景显示在左侧。

### 3.5 输入与发送

页面底部是输入框与发送按钮。用户可以在输入框中键入问题，回车或点击"发送"按钮都可以提交。提交后输入框立即清空，对话窗口先出现一条用户消息，然后系统消息以"正在思考中..."占位，等待后端返回回答后再替换为真实内容。

## 第四章 对话问答

### 4.1 提一个学业问题

以最常见的"查询学业通知"为例。用户保持在"学业 agent"标签，输入框中输入一句问题，例如"2026 年辅修招生录取名单出来了吗"，回车或点击发送。

系统先在数据库中按"学业"分类做关键词检索，命中相关通知后将这些通知的标题与摘要作为上下文，调用 DeepSeek 模型生成一段自然语言回答。回答正文显示在系统消息气泡中，前面有一个方括号包裹的话题标签例如"[学业]"，便于用户在多轮对话时快速识别。

【截图预留：请在此处插入"学业 agent 下提问并获得回答"截图。】

### 4.2 查看参考来源

每条系统回答的下方有一个"参考来源 (n)"折叠区，n 表示本次回答用到了多少条通知。点击折叠区可以展开，列出每条参考通知的标题、来源标签和原文链接。点击链接会在新标签页打开学校原文页面，便于用户核对完整通知。

参考来源列表是问答的可追溯性保障。如果系统的回答与用户的预期不符，用户可以通过参考来源直接对照学校原文，判断是检索结果偏题还是模型理解偏差。

【截图预留：请在此处插入"展开参考来源折叠区"截图。】

### 4.3 切换话题继续提问

用户可以随时点击其他话题标签切换。切换后对话窗口不会清空，新提问的回答会写入同一个对话流。例如先在学业 agent 下问辅修通知，再切到就业 agent 问"计算机学院最近有招聘吗"，系统会自动改用"就业"分类的通知做检索。

### 4.4 没有相关通知时的反馈

如果用户提问的内容在当前话题分类下没有任何相关通知，例如在党团 agent 下问"如何申请助学金"，系统会明确告知"目前知识库中没有相关内容"，并建议用户切换话题或直接查阅学校官网，不会编造不存在的政策。这是平台的设计原则之一：宁可如实告诉用户没有信息，也不允许模型臆造校园政策。

### 4.5 异常情况

如果发送提问后长时间没有收到回答，常见原因有：

- DeepSeek API Key 未配置或已欠费：系统会返回"未配置 DEEPSEEK_API_KEY，无法调用大模型"的提示
- 网络异常导致大模型接口超时：系统会返回"大模型调用失败"的提示并附带具体错误信息
- 后端服务未启动：浏览器会显示"服务暂时不可用"

遇到上述情况，用户可以检查浏览器与后端的连接，必要时联系管理员。

## 第五章 知识库浏览

### 5.1 进入知识库展示页

用户点击左侧导航栏的"RAG 知识库展示"，进入知识库浏览页。该页面提供了一种不依赖问答的浏览方式，适合用户希望按分类批量查看通知的场景。

【截图预留：请在此处插入"RAG 知识库展示页整体布局"截图。】

### 5.2 分类卡片网格

页面中央是一组分类卡片，每张卡片显示一个话题的名称（带 emoji）和当前该话题下收录的通知数量，例如"📖 学业知识库 共 142 篇 · 点击查看"。卡片数量与平台支持的话题数一致，目前为五张。

卡片下方的数字会随后台爬虫的运行自动更新。例如教务处发布了新的辅修通知，定时爬取任务把通知抓入"学业"分类后，下次刷新页面就能看到学业卡片上的数字加一。

### 5.3 查看分类下文章列表

用户点击某一张卡片，例如"📖 学业知识库"，页面下方会展示该分类下最近的若干条文章。每条文章显示标题、来源标签（例如 wechat_mp、cuc_jwc_notice）和发布日期。文章标题本身是一个链接，点击会在新标签页打开学校原文页面。

【截图预留：请在此处插入"点击学业卡片后展开的文章列表"截图。】

### 5.4 切换分类

用户可以点击任意一张其他卡片继续浏览。切换分类时，下方的文章列表会即时刷新为新分类下的内容。如果某分类下还没有文章，会显示"该分类下还没有文章"的提示，这种情况通常意味着对应的爬虫还未执行或该来源最近没有发布新通知。

## 第六章 身份设置

### 6.1 设置目的

平台支持用户保存一组基本身份信息，包括所在学院、所在年级、身份角色和补充标签。这些信息保存在浏览器本地存储中，便于后续平台根据身份做个性化推荐与展示。

### 6.2 填写步骤

用户点击左侧导航栏的"身份设置"，进入身份设置页。页面提供一个简单的表单：

1. 学院：从下拉框选择，例如"计算机学院"、"人工智能学院"、"数据科学学院"
2. 年级：从下拉框选择，例如"2024 级"
3. 身份角色：从下拉框选择，例如"本科生"、"研究生"、"辅导员"、"教师"
4. 标签补充：在文本框中填写自定义标签，例如"转专业、考研、学生干部"

填写完成后点击"保存身份标签"按钮，页面下方会显示"身份标签已保存，可用于后续对话 RAG 检索"。

【截图预留：请在此处插入"身份设置页填写后并保存"截图。】

### 6.3 数据存储说明

身份信息仅保存在用户当前浏览器的本地存储中，不会上传到服务器，也不会跨设备同步。用户清理浏览器数据或更换浏览器后需要重新填写。这种存储方式避免了平台为身份信息建立用户库的额外复杂度，同时也避免了不必要的隐私数据上传。

## 第七章 每日数据查看

### 7.1 用途

平台维护了一份按日聚合的统计数据，用于让管理员或运营人员查看各话题的请求情况，便于评估哪些话题被频繁问到、知识库需要扩充哪一类通知。

### 7.2 操作步骤

用户点击左侧导航栏的"每日数据"，进入每日数据页。页面顶部是一个日期选择器和"查看请求次数"按钮，下方是一张表格，列出五个话题标签的请求次数。

操作流程：

1. 在日期选择器中选定一个日期
2. 点击"查看请求次数"按钮
3. 页面下方的表格刷新为该日期下各话题的请求次数

【截图预留：请在此处插入"每日数据页选择日期并展示请求次数"截图。】

### 7.3 知识库更新日志

每日数据页下方还提供了一份知识库更新日志区域，列出最近几天知识库新增了哪些类型的通知，便于用户掌握平台的内容更新节奏。

## 第八章 后台运营

本章面向系统管理员，介绍如何运维平台的后台数据。

### 8.1 触发爬虫

平台预置了三个爬虫脚本，分别对接教务处、计算机学院和就业网。管理员在项目根目录命令行执行：

```
python -m scripts.run_crawler cuc_jwc_notice --max 5
python -m scripts.run_crawler cuc_cs_notice --max 5
python -m scripts.run_crawler cuc_career --max 5
```

每个命令会从对应来源抓取若干页通知。命令结束时会输出统计结果，包括"total（抓取总数）、inserted（新增数）、skipped（已存在跳过数）、errors（错误数）"。

【截图预留：请在此处插入"执行爬虫命令并看到完成统计"截图。】

### 8.2 查看数据库内容

管理员可以通过 Docker 进入 PostgreSQL 容器，使用 psql 客户端查看数据库内容：

```
docker exec -it zhiyuan-postgres psql -U zhiyuan -d zhiyuan
```

进入交互式终端后，可以执行 SQL 查询，例如：

```
SELECT category, COUNT(*) FROM articles GROUP BY category;
SELECT id, category, title FROM articles ORDER BY publish_time DESC LIMIT 20;
SELECT id, question, created_at FROM chat_logs ORDER BY created_at DESC LIMIT 20;
```

第一条查询统计各话题下通知数量；第二条查询查看最近 20 条通知；第三条查询查看最近 20 条问答记录。

【截图预留：请在此处插入"psql 查询通知数量与最近通知列表"截图。】

### 8.3 API 文档调试

平台启动后自带 Swagger 接口文档，访问 http://localhost:8000/docs 即可打开。文档列出所有公开接口的请求参数、响应格式和示例值，管理员可以直接在浏览器中点开"Try it out"做接口调试。

【截图预留：请在此处插入"Swagger 文档展开 /api/chat 接口"截图。】

## 第九章 注意事项与常见问题

### 9.1 API Key 配置

DeepSeek API Key 是问答功能的必备条件。如果浏览器对话页提示"未配置 DEEPSEEK_API_KEY"，管理员需要：

1. 打开项目根目录的 .env 文件
2. 在 DEEPSEEK_API_KEY= 后面填入完整的密钥
3. 保存文件
4. 重启 uvicorn 后端服务

注意保存文件后必须确认磁盘上的文件已更新，可以在命令行用 `type .env | findstr DEEPSEEK_API_KEY` 验证。

### 9.2 数据库连接

如果命令行执行 alembic upgrade head 时报错"Connection refused"，通常是 PostgreSQL 容器没有启动或还没进入健康状态。管理员可以执行 docker compose ps 查看容器状态，确认 zhiyuan-postgres 显示 healthy 后再重试。

### 9.3 跨域访问

平台默认放开了所有来源的跨域访问，方便开发调试。生产部署时需要把 .env 中的 CORS_ORIGINS 改为具体的域名列表，避免不必要的安全风险。

### 9.4 浏览器缓存

修改前端文件后如果浏览器看到的还是旧版本，可以使用 Ctrl+F5 进行强制刷新，绕过浏览器缓存。

### 9.5 爬虫被反爬

如果某个爬虫一直返回 0 条数据，通常是目标网站做了反爬限制。管理员可以等待一段时间后重试，或者更新爬虫脚本中的 User-Agent。
