## 五、软件用途

面向高校师生提供校园通知聚合、按话题浏览和基于通知的智能问答服务。
平台自动从教务处、计算机与网络空间安全学院、就业网三个数据源抓取通知，
按"学业 / 活动 / 党团 / 就业 / 行政"五个话题自动分类入库；用户在统一的
对话界面用自然语言提问，系统在对应话题分类下做检索增强后调用 DeepSeek
大语言模型生成回答，并在回答下方标注参考来源链接，便于用户跳转到学校
原文核对。

## 六、技术特点

1. **多源校园通知爬取与去重**：基于 `app/crawler/base.py` 提供的爬虫基类，
   实现教务、计算机学院、就业网三个垂直数据源的定向抓取，并按 URL 哈希
   做唯一约束保证去重入库。
2. **规则化关键词分类**：`app/crawler/classifier.py` 提供五类话题的关键词
   规则，通知入库时自动落分类标签，避免后期人工标注。
3. **基于 PostgreSQL 的检索增强问答**：`app/services/retrieval.py` 使用
   ILIKE + 中文 bi-gram 切词在关系数据库中做轻量检索，对小规模知识库
   足够，省去额外引入向量数据库的部署成本。
4. **与 DeepSeek 大语言模型对接的问答管道**：`app/api/chat.py` 串联检索
   结果与模型调用，在 Prompt 中显式列出参考通知，使模型回答可追溯。
5. **前后端一体的轻量部署**：FastAPI 单端口同时提供 API 与前端静态文件，
   无需额外配置 Nginx，便于校内小规模部署。

## 七、主要功能模块

1. 对话中心（学业 / 活动 / 党团 / 就业 / 行政 五个 Agent 标签）
2. RAG 知识库分类浏览
3. 身份设置（学院 / 年级 / 角色 / 标签，本地存储）
4. 每日数据统计
5. 后台通知爬取（教务、计算机学院、就业网）
6. 后台自动话题分类
7. 后台问答日志记录

## 八、与现有相似软件的区别

市面上常见的校园助手类软件多以"成绩查询"、"课表查询"为主，依赖教务系统
账号代理登录；智源不接入任何账号系统，仅就**公开通知**做聚合与问答，因此
不涉及学生个人隐私信息；同时通过 RAG 机制保证回答**有出处可追溯**，避免
大模型臆造校园政策。
