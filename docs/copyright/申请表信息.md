## 五、软件用途

面向高校师生提供校园通知聚合、按话题浏览和基于通知的智能问答服务。
平台通过 APScheduler 每 30 分钟自动巡查教务处、计算机与网络空间安全学院、
就业网三个数据源，把通知按"学业 / 活动 / 党团 / 就业 / 行政"五个话题
自动分类入库；新通知入库时同步生成 1024 维 embedding，存入 pgvector 向量
列。用户在统一的对话界面用自然语言提问，系统先把问题编码为向量，在对应
话题分类下按"余弦相似度 × 时间衰减"加权检索，得到的参考通知再交给 DeepSeek
大语言模型生成回答，并在回答下方标注参考来源链接，便于用户跳转到学校
原文核对。

## 六、技术特点

1. **多源校园通知定时自动爬取与去重**：基于 `app/crawler/base.py` 的
   爬虫基类与 `app/scheduler.py` 的 APScheduler IntervalTrigger，三个 spider
   每 30 分钟一次错峰巡查；URL 唯一约束 + SHA-256 内容哈希双层去重。
   每次跑批写入 crawl_runs 表一条流水（含开始/结束时间、status、抓取数、
   新增数、跳过数、失败数），可在前端"每日数据"页实时查看。
2. **规则化关键词分类**：`app/crawler/classifier.py` 提供五类话题的关键词
   规则，通知入库时自动落分类标签，避免后期人工标注。
3. **基于 pgvector 的向量检索增强问答**：`app/services/retrieval.py` 主路径
   使用 pgvector 余弦距离运算符 `embedding <=> query`，配合 IVFFLAT 索引；
   `app/services/embedding.py` 抽象 EmbeddingClient 接口，支持智谱 BigModel
   embedding-3 与 OpenAI 兼容 provider；不可用时静默降级为关键词 ILIKE +
   中文 bi-gram 检索，保证服务连续。
4. **时间衰减加权排序**：检索打分公式
   `score = cos_sim × exp(-lambda × age_days)`，λ 由 settings 配置，
   默认 0.05 对应半衰期约 14 天；同等相关性下偏好较新通知，
   契合高校通知"时效性高"的特点。
5. **与 DeepSeek 大语言模型对接的问答管道**：`app/api/chat.py` 串联检索
   结果与模型调用，在 Prompt 中显式列出参考通知，使模型回答可追溯。
6. **前后端一体的轻量部署**：FastAPI 单端口同时提供 API 与前端静态文件，
   无需额外配置 Nginx，便于校内小规模部署。

## 七、主要功能模块

1. 对话中心（学业 / 活动 / 党团 / 就业 / 行政 五个 Agent 标签）
2. RAG 知识库分类浏览
3. 身份设置（学院 / 年级 / 角色 / 标签，本地存储）
4. 每日数据统计（当日概览 + 分爬虫详情 + 跑批流水）
5. 后台定时爬取（教务、计算机学院、就业网；每 30 分钟一次）
6. 后台自动话题分类
7. 后台 embedding 同步与回填
8. 后台问答日志记录

## 八、与现有相似软件的区别

市面上常见的校园助手类软件多以"成绩查询"、"课表查询"为主，依赖教务系统
账号代理登录；智源不接入任何账号系统，仅就**公开通知**做聚合与问答，因此
不涉及学生个人隐私信息；同时通过向量化 + 时间衰减的 RAG 机制保证回答
**有出处可追溯且偏好最新内容**，避免大模型臆造校园政策，也避免普通关键词
检索把过期通知排在最前的问题。
